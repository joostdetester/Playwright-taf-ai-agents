Doel: op basis van input (video, OpenAPI JSON, DB-context) eerst een prompt maken, en daarna met Copilot Chat de feature files, step definitions en page objects laten genereren. Dit is de “as-is” werkwijze in deze repo.

Quick start (1x)

  1. Zet je project-ENV in .env (zie .env.example) en vul minimaal BASE_URL waar nodig.
  2. Installeer dependencies en browsers:
npm ci
npx playwright install
  1. Genereer Playwright tests uit Gherkin:
npm run bddgen
  1. Run tests (incl. Allure results):
npm run bdd
  1. Bekijk Allure report:
npm run allure:generate
npm run allure:open

Aanpak integratie testen (API + DB)

Voorbeeld in repo: bookstore DB steps + gedeelde API steps patterns.

  • DB feature: features/bookstore/db/bookstore-get-books-from-database.feature
  • DB steps: steps/bookstore/db/db.steps.ts (mysql2/promise + retry + docker bootstrap)
  • Prompt voorbeeld (API ↔ DB vergelijking): ai/prompts/api/get-books-by-author.prompt.md
  1. Zorg dat DB-ENV is gezet (lokaal via .env). De repo leest dit via config/project.config.ts.
  2. Beschrijf je integratie-flow in Gherkin als expliciete cross-layer stappen, bijv.:
    • When I create book via API
    • Then the database should contain that book
  3. Laat Copilot Chat steps genereren die:
    • API calls doen met Playwright request
    • DB verify doen met mysql client (zoals bookstore)
    • shared state opslaan op world (pattern in gedeelde steps)

Conventies (belangrijk)

  • Gebruik kebab-case voor flow/scenario namen (geen spaties).
  • Folder structuur: features/<sut>/{api,db,ui,e2e} en steps/<sut>/{api,db,ui,e2e}.
  • Gherkin steps in English, business-georiënteerd, concreet (geen “click x”).
  • Selectors: niet gokken uit video; gebruik placeholders (bijv. data-test-id) en vul later in de POM in.

Troubleshooting

  • “No tests found”: check of je npm run bddgen hebt gedraaid; Playwright runt de gegenereerde specs onder .features-gen/.
  • Scenario Outline titels in Allure: voeg <kolom> placeholders toe in de titel of gebruik title-format (zie ai/playwright-bdd-style.md).
  • Flaky UI: stabiliseer locators/waits in de page object, niet in Gherkin.

Meer detail: zie ai/video-to-test-readme.md, ai/ai-instructions.md en ai/playwright-bdd-style.md.
